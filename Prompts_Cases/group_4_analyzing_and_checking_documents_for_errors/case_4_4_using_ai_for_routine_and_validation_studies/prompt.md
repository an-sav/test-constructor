# Промпт для создания описания процесса валидации микробиологических методов





## Контекст (Context)



Создать структурированное описание процесса валидации микробиологических методов для фармацевтического производства в соответствии с [УКАЗАТЬ НОМЕР СОП] и [УКАЗАТЬ НОМЕР МЕТОДОЛОГИЧЕСКОГО ДОКУМЕНТА].



## Роли (Roles)



- Исполнитель: Специалист по [ТЕКСТ]
- Целевая аудитория: Сотрудники [ТЕКСТ]
- Ответственные лица: Руководитель лаборатории [ТЕКСТ], QA специалист [ТЕКСТ]


## Действия (Actions)



Этап 1. Извлечь из СОП:

- Критерии приемлемости для каждого типа испытаний
- Требования к документации
- Схемы проведения испытаний
- Формулы расчетов
Этап 2. Включить из методологического документа следующие этапы:
1. Подготовительный этап (определение цели, формирование группы, выбор параметров, разработка протокола).
2. Подготовка материалов и оборудования.
3. ЗАДАЧА валидации (подготовка образцов, проведение испытаний, анализ результатов).
4. Проведение основных валидационных испытаний (специфичность, предел обнаружения, линейность, правильность, прецизионность, устойчивость).
5. Обработка и анализ результатов (статистическая обработка, сравнение с критериями).
6. Документирование результатов (подготовка и утверждение отчета).
7. Заключительный этап (обновление перечня методик, актуализация инструкций, обучение).
8. Мониторинг и поддержание валидационного статуса (ревалидация).

Этап 3. Запроси необходимые процедуры проведения испытаний.



## Формат (Format)



Структурировать информацию в следующем порядке:

1. Общие требования
2. Подробный план валидации
3. Требования к материалам и оборудованию
4. Статистическая обработка
5. График валидации/ревалидации


## Технические требования (Technical)



- Вывод результата на русском языке.
- Использовать точные значения из СОП.
- Включать ссылки на соответствующие разделы документов.
- Добавить блок-схемы процессов.
- Оформить таблицы критериев приемлемости по требованиям.
- Проверить результат согласно прикрепленному документу.
